# Задача RPN: Обратная польская запись (90 баллов)

**Имя входного файла:** input.dat  
**Имя выходного файла:** output.dat  

**Ограничение времени:** 1 с  
**Ограничение реального времени:** 5 с  
**Ограничение памяти:** 64M  

---

Дано определение обратной польской записи (ОПЗ) арифметического выражения. ОПЗ может быть:

- Либо числом, например, `1`, `15`;
- Либо арифметической операцией над двумя другими выражениями в польской записи (обозначим их `A` и `B`): например, `A B +`, `A B -`.  
  В этом случае сначала записывается первый аргумент операции, потом второй аргумент операции, потом сама операция.

Рассмотрим способ представления в бинарном файле ОПЗ над целыми неотрицательными числами (размером не более 7 десятичных разрядов) и операциями `+`, `-`:

- Число в файле представляется в виде 32-битного числа (порядок байт - Big Endian, «старший разряд по младшему адресу»). В рамках 32-битного числа на 7 младших десятичных разрядов отводится ровно 4 бита, а на старший разряд - 3 бита. Если нумеровать биты 32-битного числа от младшего (бит 0) к старшему (бит 31), то десятичные разряды будут расположены (от младшего к старшему) в битах 0-3, 4-7, 8-11, 12-15, 16-19, 20-23, 24-27, 28-30. Бит 31 в числе равен нулю.
- На представление знака «+» или «-» отводится один байт. Знак «+» представляется как  `0x80`. Знак «-» представляется как `0x81`.

Требуется считать из бинарного файла `input.dat` представление выражения в ОПЗ, вычислить значение выражения и вывести результат в текстовый файл `output.dat`. В текстовом файле должно содержаться только число и перевод строки после него. Никаких других пробельных символов допускаться не должно.

Допустимо считать, что содержимое входного файла заранее было проверено на предмет соответствия формату.

Также допустимо считать, что выражение из ОПЗ содержит не более 1000 операций.

Скачать примеры бинарных файлов и условия можно по этой ссылке.

## Примеры

### Входные данные в файле input.dat

```
0x00 0x00 0x00 0x10
```

### Результат работы в файле output.dat

```
10
```

---

### Входные данные в файле input.dat

```
0x00 0x00 0x00 0x10 0x00 0x00 0x00 0x10 0x80
```

### Результат работы в файле output.dat

```
20
```

---

### Входные данные в файле input.dat

```
0x00 0x00 0x00 0x10 0x00 0x00 0x00 0x20 0x00 0x00 0x01 0x00 0x80 0x81
```

### Результат работы в файле output.dat

```
-110
```
