# Задача Generative: Изобретательный студент (75 баллов)

**Ограничение времени:** 1 с  
**Ограничение реального времени:** 5 с  
**Ограничение памяти:** 64М  

---

Студент первого курса Марк очень любит программировать. Как и большинству программистов, ему хорошо дается математика, а вот его творческая сторона хромает. Хромает настолько, что даже придумать имена для новых переменных Марку непросто.

У него есть три любимых имени для переменных:

- \( p \), потому что это первая буква слова *pointer*, а Марк очень любит указатели;
- \( i \), потому что это первая буква слова *iteration*, а Марк очень любит циклы;
- \( b \), потому что это вторая буква алфавита, а Марк очень любит разнообразие.

Вконец погрузившись в пучину творческого кризиса, Марк решил написать генератор случайных имён переменных. Помогите ему в этом, но учтите, что если в каждой программе не будет трёх его любимых переменных, Марк не сможет спокойно спать по ночам.

Напишите функцию `char* generate(int seed)`, которая генерирует имя для двух переменных по следующему алгоритму:

Занумеруем строчные буквы латинского алфавита от 0 до 25. Начиная с буквы \( p \) построим последовательность букв, в которой каждый следующий элемент получается из предыдущего путем прибавления к его номеру числа \( seed \) по модулю 26. Именем первой переменной выберем подпоследовательность букв между первым вхождением буквы \( p \) и первым вхождением буквы \( i \), а именем второй — между первым вхождением \( i \) и первым вхождением \( b \) (подпоследовательности могут пересекаться). Функция должна вернуть указатель на нуль-терминированную строку, содержащую эти имена, разделённые пробелом (даже если имена состоят из 0 букв). Если в последовательности отсутствует какая-либо из букв \( p \), \( i \), \( b \), функция возвращает нулевой указатель вместо строки.

Возвращаемая функцией строка должна быть выделена в динамической памяти. Вся память, выделенная для вспомогательных целей, должна быть освобождена до возврата из функции. По окончании работы функции должно быть выделено ровно столько памяти, сколько необходимо для корректного хранения полученного результата. Решения, содержащие ошибки в работе с памятью, получают статус *Run-time error / Ошибка выполнения*.

## Оформление решения

В этой задаче требуется написать только функцию, а не всю программу. Файл-посылка должен содержать исходную функцию, подключение необходимых заголовочных файлов, вспомогательные функции (если есть). Выводить на экран ничего не нужно, проверяться будет возвращаемое значение функции. Наличие и содержимое функции `main` в файле будет игнорироваться.

### Шаблон файла-посылки:

```c
#include <stdlib.h>

char* generate(int seed) {
    // реализация функции
}
```

*Подсказка:* можно написать всю программу (с функцией main) целиком, отладить её, а перед отправкой решения удалить или закомментировать функцию main.

## Примеры

Входные данные:

```text
15
```

Результат работы:
```text
et xm
```
